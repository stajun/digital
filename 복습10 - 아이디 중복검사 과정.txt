아이디 중복 검사 과정(github 문서 참고)
- /main/signup.jsp
  - 아이디 중복 검사 버튼을 생성
  - 중복 검사 버튼을 클릭하는 이벤트 등록
     - 입력한 아이디 값을 가져옴
     - ajax를 이용하여 아이디를 서버로 전달
        - url : /dup
        - method : post
     - 서버에서 응답한 데이터를 이용하여 
       '가입 가능한 아이디입니다' 또는
       '가입된 아이디입니다'를 알림창으로 알려줌
     - 서버에서 보내준 데이터가 회원임을 알려주면 
       아이디 중복검사를 하지 않은것으로 판별하고,
       회원이 아님을 알려주면 
       아이디 중복 검사를 한것으로 판별한다.
  - form태그의 submit 이벤트를 등록
     - 아이디 중복검사를 한 아이디만 회원가입 가능하도록 작성
  - 아이디창에 change 이벤트를 등록
     - 아이디 창의 값이 바뀌면 아이디 중복검사를 하지 않은것으로
        판별

- HomeController
  - /dup를 처리하는 메소드 추가
    - 화면에서 ajax로 넘겨준 아이디값을 가져옴(매개변수)
    - 서비스에게 아이디를 알려주면서 회원 정보를 가져오라고 요청
    - 가져온 회원정보가 있으면 화면에 회원이라고 알려주고,
      회원이 아니면 회원이 아니라고 알려줌
